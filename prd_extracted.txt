1. PRODUCT OVERVIEW
Samsung Notes is a rich note-taking app exclusive to Samsung Galaxy devices. It combines handwriting, typed text, voice memos, sketching, and document annotation in a single unified interface. The app is heavily used by online educators, students, and professionals who need versatile note capture on Android. This PRD defines a complete, open-platform replica with full feature parity plus modern cross-platform export.
1.1 Design Philosophy
Minimal, clean UI — no clutter; ink-first UX
Low-latency canvas rendering — feels like real pen on paper
Works offline-first; cloud sync is additive, not required
Modular architecture so features compose without coupling
1.2 Target Users
Online educators recording digital lessons with annotations
Students taking class notes with mixed handwriting and text
Business professionals reviewing and marking up PDFs
Creatives needing sketch + text combination boards

4. DATA MODEL
4.1 Core Entities (Room Schema)
Note { id, title, content_text, folder_id, color_tag, created_at, updated_at, is_deleted, deleted_at, is_locked, is_pinned, has_drawing, has_audio, has_image, thumbnail_path }
Folder { id, name, color, icon, parent_id, created_at }
Stroke { id, note_id, page_index, path_data(JSON), color, width, opacity, tool_type, z_index, timestamp }
AudioRecord { id, note_id, file_path, duration_ms, created_at, waveform_data(blob) }
NoteImage { id, note_id, file_path, position_x, position_y, width, height, caption }
Tag { id, name, color }  +  NoteTagCross { note_id, tag_id }
DeletedNote { id, note_id, snapshot_json, deleted_at, expire_at }
4.2 Stroke Data Format (JSON)
{ &quot;points&quot;: [[x,y,p,t], ...], &quot;tool&quot;: &quot;pen|brush|marker|eraser&quot;, &quot;color&quot;: &quot;#FF0000&quot;, &quot;width&quot;: 3.5, &quot;opacity&quot;: 0.85, &quot;smoothed&quot;: true }
Where p = pressure (0.0–1.0), t = timestamp offset in ms from recording start.
5. ARCHITECTURE
5.1 Clean Architecture Layers
Presentation: Jetpack Compose / XML Views + ViewModels (MVVM)
Domain: Use Cases (CreateNote, SearchNotes, ExportNote, etc.)
Data: Repository implementations, Room DAOs, File Storage Manager
Framework: Android OS APIs, Firebase, ML Kit adapters
5.2 Canvas Architecture
CanvasViewModel holds stroke list as StateFlow&lt;List&lt;Stroke&gt;&gt;
Custom View consumes strokes and renders via Canvas.drawPath()
OnTouchEvent feeds MotionEvent into StrokeProcessor
StrokeProcessor applies Bezier smoothing then emits to ViewModel
RenderThread offloads background layer; UI thread handles active stroke only
Tiles cache: large canvas split into 512x512px tiles, only dirty tiles re-rendered
5.3 Performance Targets
Stroke render latency: &lt; 16ms (60fps) on mid-range device, &lt; 8ms (120fps) on flagship
App cold start: &lt; 1.5 seconds
Note open (text): &lt; 300ms
PDF page render: &lt; 400ms for first page, lazy load rest
Search results: &lt; 100ms for FTS query on 1000+ notes
6. PHASE-BY-PHASE BUILD PLAN
Use these phase descriptions as direct prompts for an AI coding IDE (like Cursor, Windsurf, or AntiGravity). Each phase is a self-contained vertical slice.
PHASE 1: 
FOUNDATION — Note List + Basic Text Editor
Prompt for IDE:
Build an Android app in Kotlin with MVVM architecture using Hilt for DI and Room for local storage. Create a Note entity with fields: id (UUID), title, content, created_at, updated_at, is_deleted (soft delete), color_tag, is_pinned. Implement NoteDao with CRUD + FTS4 search. Create NoteRepository. Build HomeScreen with RecyclerView showing note cards (grid/list toggle). Each card shows title, first 2 lines of content, date, color tag dot. Add FAB to create new note. NoteEditorActivity has a plain EditText for content with a title field. Auto-save on every text change with 2-second debounce using coroutines. Support undo/redo via EditText history. Add basic toolbar: Bold, Italic, Underline using SpannableStringBuilder. Include swipe-to-delete on list with soft-delete. Add a Recently Deleted screen that restores or permanently deletes after 30 days.
PHASE 2: 
ORGANIZATION — Folders, Tags, Search, Sort
Prompt for IDE:
Extend the Phase 1 app. Add a Folder entity (id, name, color, icon, parent_folder_id). Add a NavigationDrawer with folder list. Implement drag-and-drop to move notes into folders using ItemTouchHelper. Add color tag picker (8 colors) in note editor bottom sheet. Implement multi-select mode on long press (show checkboxes, bulk action toolbar: delete/move/tag/share). Add a search bar at top of HomeScreen using Room FTS4 — highlight matching terms in results with SpannableString. Add sort menu: by date modified, date created, title A-Z, color. Add Smart Folders: Starred (is_pinned=true), All Notes, Recently Deleted. Persist last used folder and sort preference in DataStore.
PHASE 3: 
DRAWING ENGINE — Handwriting + Sketching Canvas
This is the hardest phase. Allocate the most time here.
Prompt for IDE:
Add a full drawing canvas to the app. Create a DrawingCanvasView (custom View). In onTouchEvent, collect MotionEvent MOVE points including getX, getY, and getPressure. Apply Catmull-Rom spline smoothing to convert raw points to a smooth Path. Render active stroke on UI thread in onDraw using Canvas.drawPath(). When stroke is lifted (ACTION_UP), commit it to a StrokeViewModel as an immutable Stroke object (color, width, path data serialized as JSON array of [x,y,pressure]). Support tools: Pen (round cap), Highlighter (square cap, 50% alpha, OVERLAY blend), Eraser (two modes: stroke-delete by hit-test, and bitmap erase). Add a tool picker bottom bar: pen, highlighter, eraser, shape, text. Color picker: 72-color grid. Width slider 0.5px to 20px. Opacity slider. Implement Undo/Redo via Command pattern (max 50 history items stored in a Deque; serialize to disk on exit). Add canvas background options: blank white, lined, grid, dotted (drawn as repeating paths, not images). Implement Palm Rejection: in onTouchEvent check event.getToolType() and ignore TOOL_TYPE_FINGER touches when last event from TOOL_TYPE_STYLUS was under 300ms. Add pinch-zoom and pan to the canvas using a Matrix transform — maintain a viewport Matrix applied in onDraw and transformed before hit-testing strokes.
PHASE 4: 
MIXED MODE — Text + Ink in Same Document
Prompt for IDE:
Implement a mixed note editor that combines a scrollable text editor and an ink canvas overlay. The document has pages. Each page has: a text layer (a custom EditText-derived view), an ink layer (DrawingCanvasView from Phase 3) positioned absolutely on top. User can toggle between text mode and draw mode via a floating toggle button. In text mode, touch events go to EditText; in draw mode, they go to the canvas. Support inserting image blocks inline: tap an image button, pick from gallery or camera (use CameraX), insert as a fixed-width ImageView within the document flow. Images can be resized by dragging corner handles (implement ResizableImageView). Allow adding floating text boxes on the canvas layer: tap in draw mode selects text tool, creates a draggable/resizable EditText overlay. Render the combined page (text + ink + images) to a single Bitmap for thumbnail generation (used on the note card) by calling View.draw() on each layer and compositing with Canvas.drawBitmap.
PHASE 5: 
PDF IMPORT + ANNOTATION
Prompt for IDE:
Implement PDF import and annotation. Use Android&apos;s PdfRenderer API to render PDF pages. Build a PdfViewerActivity with a vertical RecyclerView where each item is one page. Each page is rendered at 2x device pixel density using PdfRenderer.Page.render() into a Bitmap, displayed in an ImageView. Add a DrawingCanvasView overlay on top of each page view (from Phase 3). Annotations (strokes, text stamps) are stored per page in the database as Stroke objects with a page_index field. Support highlight mode: detect taps/drags over rendered text regions (use PDFBox Android port for text coordinate extraction), draw semi-transparent yellow rectangle overlays. Add a page thumbnail strip at bottom for navigation. Implement zoom + pan per page using PhotoView or a custom ScaleGestureDetector solution. Add Export Annotated PDF: use iText7 to open the original PDF, for each page get the annotation bitmap (render DrawingCanvasView to bitmap), overlay it as a transparent image layer on that page, then save to a new file. Provide Share intent for the exported PDF.
PHASE 6: 
VOICE MEMOS + AUDIO NOTES
Prompt for IDE:
Add voice memo capability. Create an AudioRecord entity. In the note editor, add a microphone FAB. Tapping starts recording using MediaRecorder (output format AAC, encoder AAC_ELD for quality). Show a recording bottom sheet with a live waveform visualizer (use AudioRecord + Visualizer API, draw bars in a custom View, animate with ValueAnimator at 30fps). Show elapsed time. Stop recording saves the file to app-internal storage and inserts AudioRecord row in Room. In the note editor, audio notes appear as a compact card showing waveform thumbnail, duration, play button. Playback uses MediaPlayer. Show a horizontal seek bar mapped to the waveform; tapping scrubs to that position. Run recording in a Foreground Service with a notification so it continues when app is backgrounded. Add delete action on audio card with confirmation dialog. Store waveform data (downsampled to 200 amplitude values) as a blob in AudioRecord for offline waveform rendering.
PHASE 7: 
IMPORT / EXPORT + SHARING
Prompt for IDE:
Implement comprehensive import/export. Export to PDF: use Android PrintManager with a custom PrintDocumentAdapter that renders each page (text layer + ink layer composited to bitmap) — this gives system-quality PDF. Export to PNG: call View.draw() on the composited note view, save as PNG via Bitmap.compress(). Export to plain text: strip formatting, save as .txt. Export to .snote backup format: create a ZIP archive (using ZipOutputStream) containing note metadata as note.json, stroke data as strokes.json, audio files, and image files. Import .snote: unzip, parse JSON, re-insert into Room, copy media files to internal storage. Bulk export: allow selecting multiple notes on HomeScreen, generate a ZIP of all their exports. Share intent: after export, use FileProvider to expose the file, then fire Intent.ACTION_SEND with correct MIME type so user can share to Drive/email/WhatsApp etc. Add a dedicated Export screen with format picker and destination picker (save to Downloads vs. share). Handle large exports in a WorkManager task with progress notification.
PHASE 8: 
SECURITY, POLISH, PERFORMANCE
Prompt for IDE:
Add final polish, security, and performance optimizations. Note Locking: add a lock toggle in note options menu. Locked notes show a lock icon on the card and a blurred thumbnail. Opening a locked note triggers BiometricPrompt (fingerprint/face/PIN fallback). Store a boolean is_locked in Room; encrypt the note content using Android Keystore AES-GCM with a note-specific key derived from the note ID. Dark/Light theme: implement full Material 3 theming with a dark and light variant. All colors defined in theme XML with night-mode equivalents. Home screen widget: implement a 2x2 AppWidget using RemoteViews showing recent 3 notes as text; tapping opens the note. Performance: profile with Android Profiler and fix any jank. Ensure DrawingCanvasView onDraw is called on RenderThread by using hardware layer. Implement lazy bitmap tile loading for large canvases. Add RecyclerView prefetch for note list. Use DiffUtil for all RecyclerView updates. Add a first-run onboarding walkthrough (3 screens using ViewPager2) explaining the 3 key modes: Text, Drawing, PDF.
